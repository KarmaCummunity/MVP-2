# Audit 2026-05-16 — Auth & Profile

## Summary

Audit of FR-AUTH-* and FR-PROFILE-* surfaces against current code on `dev`. Twenty findings: 2 🔴 (sign-out leaves push token registered, mapUserRow privacy_mode cast accepts undeclared values), 11 🟠 (AC misses with user-visible impact: welcome screen has no "Sign up / Log in" entry per FR-AUTH-001 AC2, soft-gate wired only to post-create, missing FR-AUTH-018 AC3 Back, avatar resize edge mismatch, FR-PROFILE-015 unimplemented, followers list non-paginated, forgot-password dead button, SignInWithEmail leaks `invalid_email` distinct from `authentication_failed`, OnLockPress tappable on other-user profile, biography URL regex over-matches innocent compounds, edit-profile lacks read-only email/phone block), 7 🟢 (raw-Error throws, ResendVerificationEmail no rate-limit enforced in use case, missing Share Profile, etc.). Re-confirmed still-open items from the 2026-05-10 follow-up.

- **§5.7** (FR-AUTH-011 AC4 avatar-removal Storage leak) — partially addressed by `removeUploadedAvatar` (TD-108) in edit-profile + onboarding photo flows, but the SSO-prefilled avatar imported by `handle_new_user` (`migration 0084` line 109) is the raw Google CDN URL — it is never copied to our bucket and `removeUploadedAvatar` deletes nothing for those users.
- **§5.10** (FR-PROFILE-007 AC2 read-only email/phone on Edit Profile) — still open; see AUTH-11 below.
- **§6.3** (TD-106 Edit Profile share button no-op) — still open; `MyProfileChrome.tsx:110` `<TouchableOpacity style={styles.shareBtn}>` has no `onPress`.
- **§14.7** (TD-24 + TD-151 Apple SSO + Phone OTP placeholder) — still open; no Apple or phone OTP infra exists anywhere in `packages/infrastructure-supabase/src/auth/`. The iOS welcome "Apple" button routes to `/(auth)/sign-in` (email/password).

## Spec amendments needed

- **FR-AUTH-001 AC2 vs spec status header.** The status header notes "email/password via secondary link on welcome." but AC2 says navigation to "the auth screen (FR-AUTH-002)" which on Android/Web is supposed to be a tabbed picker. The two are now divergent; recommend documenting that the MVP welcome only ships Google + Guest + Apple-shell and call the missing email link a TD-151 sub-row.
- **FR-AUTH-011 AC2.** Code resizes to 512 (`AVATAR_RESIZE_EDGE`). Either bring code to 1024 or amend AC2 to 512 (likely the right call for MVP — 1024 JPEG q=85 is ~150 KB per avatar; 512 is plenty).
- **FR-AUTH-018 vs FR-AUTH-010.** Spec re-numbers from `FR-AUTH-018` (mini about, step 1) → `FR-AUTH-010` (basic info, step 2) → `FR-AUTH-011` (photo, step 3) → `FR-AUTH-012` (tour, step 4). The implementation's `OnboardingStepHeader` step indicator follows the 1-2-3-4 ordering but the FR-ID jump from 018→010 is confusing in code review. Consider renumbering FR-AUTH-018 → FR-AUTH-009.5 or noting the chronology in the spec preamble.
